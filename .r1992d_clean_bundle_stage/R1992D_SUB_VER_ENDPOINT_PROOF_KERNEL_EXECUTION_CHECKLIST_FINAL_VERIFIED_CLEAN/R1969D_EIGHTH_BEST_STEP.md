# R1969D eighth best step: formalization priority and proof-burden ordering

R1969D adds a formalization-priority layer above the R1968D anchor-classification layer.

The purpose is to distinguish, in Lean-checkable bookkeeping form, which remaining analytic anchors are:

- immediate core new analytic formalization tasks,
- high-priority external theorem integration tasks,
- high-priority manuscript-bridge tasks,
- medium-priority library or routine formalization tasks, or
- ledger/audit-only tasks.

This layer does not prove the concrete Li kernel, the explicit formula, the packet estimates, the Li criterion, or RH.  It only orders the remaining proof burden while preserving the already verified bridge:

formalization priority plan
→ anchor classification
→ source evidence slots
→ concrete source categories
→ realization interfaces
→ micro-targets
→ analytic decomposition slots
→ analytic realization obligations
→ R1961D packet-balance layer
→ actual Li target.
